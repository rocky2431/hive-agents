## Design Context

### Users
Enterprise managers and business teams — primarily non-technical people who configure and interact with AI "digital employees" daily. They care about getting work done through agents, not about the underlying tech. Their context: managing agent teams, reviewing agent output, configuring workflows, and chatting with agents across channels (Feishu, Slack, etc.). The interface must feel approachable enough that a marketing manager or HR lead can confidently set up and manage AI agents without developer support.

### Brand Personality
**Intelligent · Cutting-edge · Refined** (智能 · 前沿 · 精致)

Like Vercel and Raycast — sophisticated tech-forward feel that signals powerful capability without being intimidating. The interface should whisper "this is serious technology" through precision and polish, not through complexity. Every interaction should feel intentional and considered.

**Emotional goals:** Confidence (I trust this tool with real work), competence (this was built by people who care about craft), and clarity (I always know what's happening and what to do next).

### Aesthetic Direction
**Visual tone:** Clean, bright, and collaborative — inspired by Notion and Slack's warmth and approachability, elevated with the refined precision of Vercel/Raycast.

**References:**
- **Notion** — Clean whitespace, friendly but professional, strong content hierarchy, collaborative feel
- **Slack** — Warm personality, approachable communication UI, clear status indicators, good use of color as information

**Anti-references:**
- Overly dark/hacker aesthetic (current theme leans too Linear-dark for the target audience)
- Dense enterprise dashboards with too many controls visible at once
- Generic SaaS templates with no personality

**Theme:** Dark and light mode both supported. Light mode should be the primary experience for enterprise/business users (Notion/Slack convention). Dark mode as a polished alternative.

**Current state:** Linear-inspired dark-first design with customizable accent colors, Inter font, 50+ CSS variables. The foundation (token system, spacing scale, typography) is solid. Direction should evolve toward brighter, warmer, more Notion/Slack-aligned defaults while keeping the refined quality.

### Design Principles

1. **Clarity over cleverness** — Every element earns its space. Prefer clear labels, obvious affordances, and predictable patterns over novel interactions. Business users should never wonder "what does this do?"

2. **Warm intelligence** — Balance the tech-forward brand with approachable warmth. Use color purposefully (status, categories, personality), generous whitespace, and friendly micro-copy. AI should feel like a capable colleague, not a cold system.

3. **Progressive disclosure** — Show the simple path first, reveal power on demand. Agent configuration should feel as easy as filling out a profile, with advanced options accessible but not overwhelming.

4. **Information density when it matters** — Dashboards and agent management views should be scannable and data-rich (Linear's density). Chat and onboarding flows should breathe (Notion's spaciousness). Match density to context.

5. **Consistent motion, minimal animation** — Transitions should be fast (120-200ms) and purposeful. Use animation to communicate state changes and guide attention, never for decoration. Status indicators (running, idle, error) deserve the most animation budget.

### Technical Foundation (Current)
- **CSS Architecture:** Single global `index.css` (1,730 LOC), CSS custom properties, no framework
- **Typography:** Inter (300-700), 14px base, 7-step scale (11-32px), JetBrains Mono for code
- **Spacing:** 4px base unit (4-48px scale), sidebar 240/68px, header 52px
- **Colors:** 50+ CSS variables, 10 accent presets, dark/light via `data-theme` attribute
- **Icons:** @tabler/icons-react
- **Radius:** 4/6/8/12/9999px scale
- **Transitions:** 120ms (fast), 200ms (default), 300ms (slow)
- **i18n:** Chinese + English, LTR only
- **No component library** — all custom, BEM-inspired class naming
